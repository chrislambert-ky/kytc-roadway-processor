Kentucky Transportation Cabinet

Roadway Attribute Processor

A browser-based tool for enriching coordinate datasets with Kentucky roadway attributes from the KYTC Spatial API — no software installation required.

1
Load Data

Upload a CSV file containing point coordinates. The tool auto-detects latitude and longitude columns and shows a live spreadsheet preview. Use the column selector to hide fields you don’t need before processing.

2
Transform

Choose which KYTC roadway attributes to append to each row. A curated set of defaults (route, milepoint, district, county, etc.) are pre-selected and locked. Additional fields from the full API schema can be toggled on or off.

3
Process

The tool sends each coordinate pair to the KYTC Spatial API in parallel async batches of up to 500 requests. Progress is shown live. Rows that cannot be matched are flagged with a status code rather than silently dropped.

4
Extract

Select which columns to include in the export, then download in your preferred format: CSV, JSON, GeoJSON, KML, Excel, Parquet, or GeoParquet. A 10-row preview updates as you toggle columns.

Technical notes

  • All processing happens entirely in your browser — no data is sent to any server other than the KYTC Spatial API.
  • Large CSV files are loaded via DuckDB-WASM, an in-browser SQL engine. Parquet and GeoParquet export also use DuckDB.
  • Excel export uses SheetJS. All other formats are generated with plain JavaScript.
  • A date_processed column is automatically appended to every row recording the local date the API was queried — useful for tracking which version of the linear referencing system was used.

Sample data

Looking for data to try? Visit KYTC TRAK Data Hub and download bridge data in CSV format.
